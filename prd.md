# PRD: Vibe Coding Diary

## 背景与目标

作者是一名程序员，正在通过 vibe coding（用自然语言描述需求，借助 AI 生成技术方案并快速落地验证）加速自己成长为架构师。这个博客用来记录这段旅程中的思考、实践和收获。

**目标**：搭建一个风格宁静、阅读体验舒适的个人博客，让作者能够专注于写作，让读者能够愉快地阅读。

## 目标用户

- **作者本人**（内容创作者）：用 Markdown 写文章，偶尔需要在文章中插入交互式内容（如图表、可操作的演示）来更好地表达技术思路
- **读者**：对 vibe coding、AI 辅助开发、架构成长感兴趣的中英文开发者

## 用户场景

### 作者写作

1. 作者用 Markdown 撰写一篇新文章，填写标题、日期、摘要等基本信息
2. 文章内容以文字为主，但某些段落需要嵌入交互式组件（比如一个可切换的架构对比图）来辅助说明
3. 作者将文章文件放入指定目录，构建后自动生成对应页面，无需手动配置路由或页面模板

### 读者浏览

1. 读者首次访问博客，通过首页的视觉意象和标语感受博客氛围，了解博客主题
2. 读者进入文章列表页，浏览所有文章，找到感兴趣的内容
3. 读者打开文章详情页，阅读完整内容，与文章中的交互组件进行互动
4. 中文读者和英文读者各自看到对应语言的内容和界面
5. 读者通过 RSS 订阅博客，在阅读器中获取更新

## 功能需求

### 页面结构

#### 首页

- 顶部视觉区域：以"西湖边吃薯条的鸟"为视觉意象创建的图案。一句标语，传递博客的主题——用闲适自在的态度记录 vibe coding 的成长之路
- 最近发布的文章预览（标题 + 摘要，最多 3 篇）
- 导航至文章列表页的入口

#### 文章列表页

- 展示所有已发布的文章，每篇显示标题、发布日期、摘要
- 按发布时间倒序排列

#### 全局页脚

- 所有页面底部展示作者的 GitHub 链接

#### 文章详情页

- 展示文章标题、发布日期
- 渲染文章正文，支持标准 Markdown 语法及嵌入的交互式组件
- 返回文章列表的导航

### 内容管理

- 作者以 Markdown 编写文章，存放在固定目录下
- 每篇文章包含结构化的元信息：标题、日期、摘要、语言标识
- 构建时自动将文章转换为页面，无需手动创建页面文件
- 支持在 Markdown 中嵌入交互式组件，用于表达纯文字难以传递的内容

### 中英文双语

- 每篇文章标记所属语言
- 站点界面文案（导航、按钮、日期格式等）根据当前语言自动切换
- 用户可通过切换按钮在中英文之间切换
- 文章列表按当前语言筛选显示

### RSS 订阅

- 自动生成 RSS feed
- 包含文章的标题、摘要、日期和链接
- 浏览器和阅读器能自动发现订阅源

### 暗色模式

- 支持亮色和暗色两种主题
- 默认跟随用户系统设置
- 提供手动切换按钮，用户的选择会被记住

### 响应式设计

- 适配桌面、平板、手机
- 导航在小屏幕上折叠为菜单
- 文章卡片在小屏幕上单列排列
- 正文排版在移动端保持舒适的阅读体验

## 非功能需求

- **性能**：页面为纯静态，首屏加载快速
- **SEO**：页面结构语义化，支持搜索引擎收录
- **可维护性**：新增文章只需添加 Markdown 文件，无需修改代码

## 成功指标

- 作者能在 5 分钟内完成一篇新文章的发布（写完 Markdown → 构建 → 上线）
- 页面在移动端和桌面端均有良好的阅读体验
- Lighthouse 性能评分 ≥ 90
